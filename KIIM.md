# KIIM.md — 작업 과정 & 최종 점검 노트

HomeProtector(홈프로텍터) 프로젝트를 클론한 시점부터 진행한 작업의 과정·구조·검증·리스크를
한 곳에 정리한 문서입니다. (사용자 가이드는 [`README.md`](./README.md), API 명세는
[`ENDPOINT.md`](./ENDPOINT.md) 참고)

---

## 1. 클론 후 빠른 시작

```bash
git clone <repo-url> && cd home_protector
flutter pub get
cp .env.example .env        # 키 입력(선택). 키 없이도 실행은 됨
./run.sh                    # 웹(Chrome) http://localhost:56940
./run.sh device             # 모바일 실기기/에뮬레이터
```

`.env` 에 채울 수 있는 값: `KAKAO_JS_KEY`(웹 지도), `KMA_SERVICE_KEY`(기상청 특보/일자료),
`CORS_PROXY`(웹에서 기상청 호출용 자체 프록시). 자세한 설명은 README의 "환경 변수" 절.

## 2. 작업 과정 (무엇을, 왜 바꿨나)

처음 상태는 "광주 주월동" 기반의 목업 위주 데모였고, 외부 API와 불일치·하드코딩·디자인 이슈가
있었습니다. 다음 순서로 프로덕션화했습니다.

| 단계 | 내용 |
|---|---|
| **API 정합화** | Swagger 실측 → Ready-Flow는 **서울 93개 법정동 전용**임을 확인. 데모 지역을 커버리지 밖 광주 → **서울 관악구 신림동**(2022 침수 참사 지역)으로 전환. 타입 클라이언트 `flood_api_service.dart` 신설 |
| **주소 강제 선택** | 자유 입력 제거 → `/api/dongs` 기반 검색 모달(`dong_picker.dart`). 선택값에 `adm_cd`가 실려 예측이 404 없이 해결 |
| **위험도 실측화** | 98% 위험도의 정체였던 목업 시나리오 엔진(`WeatherDataService`) **완전 삭제**. 모든 위험 표시를 Ready-Flow 실측치로 일원화, 데이터 없으면 `—` |
| **기상특보 실연동** | 알림 탭을 기상청 기상특보 조회 API로 교체(`weather_warning_service.dart`). 상태/오류 상세 구분 |
| **캘린더 실측화** | 가짜 % 생성기 제거 → 기상청 ASOS 일자료로 이번 달 일강수량 렌더링(`kma_daily_service.dart`) |
| **리디자인** | 짙은 오렌지/빨강 + 이모지 → 모던 인디고-블루 + 라인 아이콘, 밝은 팔레트 |
| **하드코딩 정리** | 죽은 위젯/가짜 LIVE 데이터/임의 수치 제거, 실 API 없는 콘텐츠는 `예시` 배지 |
| **패키징/문서** | `.env` gitignore + `.env.example`, 웹 포트 56940 고정, README/ENDPOINT/KIIM 문서화 |

세부 변경은 git 커밋 메시지에 단계별로 기록되어 있습니다.

## 3. 외부 API 연동 (3종)

| API | 용도 | 인증 | 웹 | 모바일 |
|---|---|---|:---:|:---:|
| **Ready-Flow AI** (`/api/predict` 외) | 법정동 침수확률 | 불필요 | ✅* | ✅ |
| **기상청 기상특보** (WthrWrnInfoService) | 알림 탭 실시간 특보 | `KMA_SERVICE_KEY` | ⚠️ CORS | ✅ |
| **기상청 ASOS 일자료** (AsosDalyInfoService) | 캘린더 월간 강수량 | `KMA_SERVICE_KEY` + **별도 활용신청** | ⚠️ CORS | ✅ |

\* Ready-Flow도 ACAO 헤더가 없어 웹에서 차단될 수 있으나, 앱은 실패 시 안전하게 `—`/안내로
처리합니다. 기상청 2종은 **모바일 전용**이며 웹은 자체 CORS 프록시를 설정해야 합니다.

## 4. 실데이터 검증 결과 (라이브 fetch)

`POST /api/predict` 에 평범한 강우와 **실제 침수 사례**(2022-08-08 서울 기록적 폭우, 신림동
반지하 침수 참사 당일, 일강수량 ≈381mm)를 넣어 검증:

| 케이스 | 입력(mm, 과거→오늘) | 결과 `flood_probability` |
|---|---|---|
| 평범: 신림동 일반 강우 | `[5,15,25,35]` | **0.0 (0%)** |
| 대조: 신림동 무강우 | `[0,0,0,2]` | **0.0 (0%)** |
| **실제침수: 신림동 2022-08-08** | `[30,80,180,381]` | **0.0765 (8%)** |
| **실제침수: 강남 대치동 2022-08-08** | `[30,80,180,381]` | **0.0765 (8%)** |

응답 스키마(`gu`/`dong`/`adm_cd`/`flood_probability`)와 통신은 정상(200, 250~650ms).
→ **연동·파싱은 검증 완료.** 단, 아래 리스크 #1 참고.

## 5. 알려진 한계 & 리스크 (최종 판단)

| # | 리스크 | 영향 | 대응 |
|---|---|---|---|
| **1** | **Ready-Flow 모델이 강우 크기에 거의 무감각** — 기록적 폭우(381mm)도 8%, 평상시 0% | 헤드라인 "AI 침수확률"이 항상 한 자릿수로 낮게 표시됨 | 업스트림 모델 한계. 앱은 값을 **조작하지 않고 그대로** 표시(정직). 모델 개선 시 자동 반영 |
| **2** | 기상청 API **웹 CORS 차단** | 웹에서 특보/캘린더 불가 | **모바일 전용**으로 명확히 안내(UI). 웹은 자체 `CORS_PROXY` 설정 시 가능 |
| **3** | ASOS 일자료 **별도 활용신청 필요** | 미신청 시 캘린더 403 | UI에 "신청 필요" + 진단(HTTP 403) 안내 |
| **4** | ASOS 응답 필드(`tm`/`sumRn`)는 **키 미구독으로 라이브 미검증** | 필드명이 다르면 파싱 0건 | 문서 표준 필드 사용 + 방어적 파싱. 특보(t6/tmFc/tmSeq)는 라이브 검증 완료 |
| **5** | 자체가 아닌 **공개 CORS 프록시에 키 노출** 가능성 | 키 유출 | 공개 프록시 기본 비활성. 자체 프록시 권장(README에 Cloudflare Worker 예시) |
| **6** | 상태 **비영속**(메모, 설정, 체크리스트는 메모리) | 앱 재시작 시 초기화 | 데모 범위. 영속화는 향후 `shared_preferences` 등으로 |
| **7** | 일부 콘텐츠(주민제보·대피소·커머스) **데모** | 실데이터 아님 | `예시` 배지로 명시 |

**보안 점검:** `.env`/생성 키 파일은 gitignore, 저장소에 비밀키 없음. 웹 Kakao JS 키는 본래
공개(클라이언트) 키라 노출 정상. → 통과.
